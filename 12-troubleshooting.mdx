## License errors

<AccordionGroup>
  <Accordion title='"Could not reach license server. Check your internet connection."'>
    The bot needs live internet access to its license server on every launch — there's no offline mode. Check your connection, and check whether your network (some corporate/school/public networks) blocks outbound HTTPS requests.
  </Accordion>
  <Accordion title="The bot prints a rejection reason and exits immediately">
    The bot shows the exact reason your license server sent — read it first. Common causes:
    - The key was already activated on a different computer (see the machine-lock note in [Getting a License Key](/getting-a-license-key)).
    - The key expired (keys last 30 days from generation).
    - The key was typo'd when pasted into `authentication.license_key`.

    Re-running `/generate-license-key` in Discord refreshes your key for another 30 days and unlocks it from its previous machine.
  </Accordion>
  <Accordion title='"Your version is outdated" and the bot updates itself'>
    This is expected, not an error — the bot auto-updates when the license server says a newer version is required. Let it finish; it will restart on its own on macOS, or relaunch itself on Windows. Don't close the window or delete the executable mid-update.
  </Accordion>
</AccordionGroup>

## First run / config

<AccordionGroup>
  <Accordion title="The bot printed a message about a config file and closed — is it broken?">
    No — this is expected on the very first run. See [Installation & First Run](/installation#run-it-once). Open the newly created `lumi_mint_bot_config.yaml`, fill it in, and run the bot again.
  </Accordion>
  <Accordion title="I edited the config but nothing changed">
    Choose **↺ Reload Config** from the main menu, or fully restart the bot.
  </Accordion>
</AccordionGroup>

## Minting

<AccordionGroup>
  <Accordion title='A mint fails with something like a "price guard" error'>
    `mint.mint_price_eth` is a price **ceiling** — the bot refuses to mint if the drop's real price is higher than this value. Check the drop's actual price and raise `mint_price_eth` to match (or higher). On Transient, remember to also account for the separate protocol fee — see [Minting Guide](/minting-guide#platform-differences).
  </Accordion>
  <Accordion title="Gas Monitor mode (C) seems stuck, printing 'Base fee: X gwei — waiting...' forever">
    Gas Monitor waits for the network base fee to drop below `gas.base_gwei`. If you left this at its default (`1.0`), that threshold may be unrealistically low for the chain you're on. Set `gas.base_gwei` to a realistic value — see the warning in [Configuring the Bot](/configuration#gas-settings).
  </Accordion>
  <Accordion title="I'm seeing rapid repeated log lines like 'not open yet — <function name>'">
    This is "snipe mode," triggered when `gas.gas_limit` is `0`. The bot is deliberately retesting the transaction very quickly until the contract accepts it — useful when you're not sure of the exact open time. It's not an error. Be aware this generates real RPC traffic and can hit rate limits on free public endpoints; consider using a dedicated/paid RPC (`rpcs` in [Configuring the Bot](/configuration#rpc-endpoints)) if this happens often.
  </Accordion>
  <Accordion title="MEV Tip mode isn't landing my mint">
    "Bundle did not land within max_blocks" means your tip (`bundle.tip_eth`) likely wasn't competitive against other bots targeting the same block. Try increasing it — see [Mint Modes Explained](/mint-modes#e-mev-tip).
  </Accordion>
  <Accordion title="I set proxy.enabled but I don't think my mint transaction went through the proxy">
    That's expected — proxy settings only cover OpenSea sign-in and eligibility checks. The actual mint transaction always goes out directly, unproxied, for speed. See [Configuring the Bot](/configuration#proxy).
  </Accordion>
</AccordionGroup>

## Wallets

<AccordionGroup>
  <Accordion title="One of my wallets didn't load / isn't showing in the wallet checklist">
    Check the terminal output when the bot starts — it prints a warning and skips any wallet entry it couldn't load (bad private key format, missing or wrong-password keystore file). See [Setting Up Your Wallets](/wallet-setup) for the exact format required.
  </Accordion>
  <Accordion title="I'm being asked for a keystore password repeatedly">
    The password is cached in memory for up to 4 hours per run — if it's expired, or this is a new run, you'll be asked again. This is expected behavior, not a bug.
  </Accordion>
</AccordionGroup>

## Where to find logs

Every run creates a timestamped log file (e.g. `mint_bot_20260708_140213.log`) next to the executable. These are the first thing to check yourself, or to send along if you're asking for help — they're safe to delete afterward, and safe to delete in bulk since the bot doesn't clean them up automatically.

<Warning>
  Before sharing a log file with anyone, double check it doesn't contain a private key or license key pasted in by mistake.
</Warning>
